Entities

There are a few special entities that have special meaning in HTML that should always be encoded:

  • The ampersand: & (&amp;)
  • Less than and greater than: < and > (&lt; and &gt;)

There are also other entities that are special to XML and that should always be encoded in XML documents:

  • Double quotes: " (&quot;)
  • Single quotes: ' (&apos;)

With XML it is safer to use numbered entities if entities are used above native UTF-8 characters since older XML parsers have limited support for named entities where most old and new XML parsers support numbered entities without issue.

A short list of other helpful entities:

Named entities are limited to a small subset of Unicode characters, but numbered entities can specify any character.
